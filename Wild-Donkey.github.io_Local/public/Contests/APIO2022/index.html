T2

这道题应该是耗时最短的吧. 一条边 (u,v)(u, v)(u,v) 加入后能出现一个经过特殊点的环, 当且仅当存在一个特殊点 xxx 使得 vvv 能到 xxx 且 xxx 能到 uuu. 因为显然对于特殊点, 如果一个点能到特殊点 iii, 那么它一定可以经过一开始就存在的那些边通往编号比 iii 大的所有特殊点, 因此每个点能到达的特殊点一定是特殊点的一个后缀, 所以每个点只要记录能到达的最小特殊点的编号 MnMnMn 即可. 同样地, 每个点还要维护能到达这个点的特殊点的最大编号 MxMxMx.

因为 MxMxMx 记录的是到达这个点的信息, 所以它沿着正向边有传递性, 而 MnMnMn 则是沿反向边有传递性的. 所以对于已经存在的边 (u,v)(u, v)(u,v), 一定存在 Mnu≤MnvMn_u \leq Mn_vMnu​≤Mnv​, Mxu≤MxvMx_u \leq Mx_vMxu​≤Mxv​, 每次连边之后, 我们只要在正图中将 MxMxMx DFS 更新, 在反图中将 MnMnMn DFS 更新即可维护这两个值. 每次连边后如果 Mnv≤MxuMn_v \leq Mx_uMnv​≤Mxu​, 则出现回路.

这样做的复杂度是 O(q(n+q))O(q(n + q))O(q(n+q)) 的, 因为每次连边都会进行 DFS.

但是因为每次 DFS 一个点的时候, 它的 MnMnMn 会减小或它的 MxMxMx 会增大, 所以复杂度还和 kkk 有关, 每个点 DFS 的次数是 O(k)O(k)O(k) 的, 因此复杂度会变成 O(k(n+q))O(k(n + q))O(k(n+q)), 可以拿到 60′60'60′.

T3

这个题意比较简单, 就是构造一个尽可能短的排列, 使它本质不同的上升子序列数量恰好为给定的 kkk.

前 10′10'10′ 要求的是线性长度即可, 如果我们有一个单调下降的长为 nnn 的排列, 那么它的上升子序列有 n+1n + 1n+1 个, 其中有一个空序列, 还有 nnn 个长度为 111 的序列. 因此对于 kkk, 我们只要构造长度为 k−1k - 1k−1 的单调递减排列即可.

接下来就可以考虑倍增构造了, 一个长度为 nnn 的单调递增排列有 2n2^n2n 个子序列, 它们都是单增的, 如果我们可以把一些这样的序列合并起来, 那么就可以在 O(log⁡2)O(\log^2)O(log2) 的长度内构造出我们需要的排列了.

现在我们先不考虑空序列, 因为无论如何构造排列, 都存在一个空序列, 因此把 kkk 减 111 然后考虑非空上升子序列数量即可. 如果不考虑空序列, 那么我们可以通过把两个排列拼起来得到一个单调递增子序列数量为两者之和的新排列, 具体方法是对于两个长度为 n1n_1n1​, n2n_2n2​ 的排列, 我们把长为 n1n_1n1​ 的排列中每个元素增加 n2n_2n2​, 然后把长为 n2n_2n2​ 的排列放到长度为 n1n_1n1​ 的排列后面.

这样合并的原理是对于合并后的排列, 可以以 n1n_1n1​ 位置为界, 大于等于 n1n_1n1​ 的位置都比小于 n1n_1n1​ 的位置上的元素小, 因此它的上升子序列不可能同时包含小于 n1n_1n1​ 的元素和不小于 n1n_1n1​ 的元素, 因此它的上升子序列和原来两个排列的上升子序列一一对应.

但是因为不计空序列, 所以长度为 nnn 的上升排列的非空上升子序列是 2n−12^n - 12n−1, 一开始想的是在每个上升排列后面加一个长为 111 的排列, 共同组成一个有 2n2^n2n 个上升子序列的模块, 但是因为合并的每个排列是可交换的, 所以我们可以先按 2n2^n2n 来计算, 然后统计一共少了多少子序列, 然后一并插入对应数量的长为 111 的序列来抹零. 最终长度的复杂度为 O(log⁡2k)O(\log^2 k)O(log2k).

但是显然在 log⁡k\log klogk 比较多的时候这样做是十分愚蠢的, 因此对于这个 O(log⁡k)O(\log k)O(logk) 的尾巴, 我们可以递归在后面再构造一个对应数量的排列即可, 递归边界是 k=1k = 1k=1, 直接构造长度为 111 的排列即可. 每层递归 kkk 最多变成 log⁡k\log klogk, 因此层数为 O(α(k))O(\alpha (k))O(α(k)), 每层合并 O(log⁡k)O(\log k)O(logk) 个长度 O(log⁡k)O(\log k)O(logk) 的排列, 因此总长度的一个比较松的复杂度为 O(log⁡2kα(k))O(\log^2 k\alpha(k))O(log2kα(k)), 不过显然真正的复杂度比这个要小得多, 可以认为是 O(log⁡2k)O(\log^2 k)O(log2k), 但是没有那么容易证.

然后我们发现, 对于这种合并式的构造, 其实还有更高明的方法: 把所有 2n−12^n - 12n−1 从小到大排序, 然后倒序枚举, 只要 k≥2i−1k \geq 2^i - 1k≥2i−1, 就将一个长为 iii 的上升排列加入构造中, 然后把 kkk 减少 2i−12^i - 12i−1, 否则就将 iii 减 111, 这样是更加先进的 O(log⁡2k)O(\log^2 k)O(log2k) 做法, 复杂度不变, 但是可以拿到 71.2′71.2'71.2′.

如果想做到 O(log⁡k)O(\log k)O(logk), 肯定不能是 O(log⁡k)O(\log k)O(logk) 个排列合并起来. 现在重新考虑空序列.

如果一个含有 xxx 个上升子序列的排列全部元素加 111, 然后在后面加一个 000 作为新元素, 那么这个新排列将存在 x+1x + 1x+1 个上升子序列, 这 x+1x + 1x+1 个子序列中有 xxx 个是原排列就存在的, 第 x+1x + 1x+1 个子序列是仅包含 000 的子序列. 而只要一个子序列包含 000, 那么它便不能再包含任何其它元素了, 因为其它元素都比 000 大且在 000 前面.

如果一个含有 xxx 个上升子序列的排列全部元素加 111, 然后在前面加一个 000 作为新元素, 那么这个新排列将存在 2x2x2x 个上升子序列, 这 2x2x2x 个子序列中有 xxx 个是原排列就存在的, 后 xxx 个子序列是在原来存在的子序列的基础上, 在前面加一个 000 得到的. 因为 000 比任何元素都小, 而且最靠左, 所以 000 后面加一个上升子序列还是一个上升子序列.

所以我们可以只增加一个元素就可以实现将原排列的上升子序列数量倍增或者加 111, 所以可以直接根据 kkk 的二进制情况进行操作. 初始上升子序列数量可以是 222 或 333, 也就是长度为 111 或 222 的下降子序列, 不存在上升子序列为 111 的非空排列. 接下来, 我们从高到低扫描 kkk 的二进制表示, 如果遇到 000, 那么直接倍增上升子序列数量, 否则就先倍增, 然后加 111.

如果 kkk 的二进制表示长度为 aaa, 并且有 bbb 个 111, 那么我们构造的序列长度就是 a+b−1a + b - 1a+b−1, 因为 aaa 和 bbb 都是 log⁡k\log klogk 级别的, 所以构造的排列长度是 O(log⁡k)O(\log k)O(logk) 的.
